🎯 2-Tier 아키텍처 선택 이유

Tier 1 (라즈베리파이): 저전력 24/7 크롤링, 분산 처리로 속도 향상, 데이터 수집 전문화
Tier 2 (외부 서버): 고성능 AI 분석, 빠른 대시보드 응답, 개발 환경 유연성
결과: 단일 시스템 대비 과부하 방지, 확장성 확보, 전력 소비 최소화

🔄 완전 자동화 데이터 파이프라인

1

📡 데이터 수집 (Crawling)

5개 사이트에서 Scrapy Spider가 5~30분 주기로 자동 크롤링. 일일 500~2,500개 데이터 수집.

2

🧹 데이터 정제 (Cleaning)

HTML 태그 제거, 중복 제거, 데이터 타입 표준화, 이상치 필터링.

3

💾 분산 저장 (HDFS)

Hadoop HDFS에 원본 데이터 분산 저장. 3중 복제로 안정성 확보.

4

🗃️ DB 통합 (MariaDB)

정제된 데이터를 MariaDB에 저장. 인덱싱으로 빠른 쿼리 지원.

5

🤖 AI 감성 분석 (FinBERT)

뉴스 텍스트를 FinBERT 모델로 분석하여 감성 스코어 (-1~+1) 산출.

6

📊 대시보드 시각화

React 대시보드에서 실시간 차트, 알림, 포트폴리오 추천 제공.

⚡ 전체 파이프라인 처리 시간: 평균 2~5분 (수집부터 분석까지)
🔄 업데이트 주기: 뉴스 5~15분, 가격 1분, Fear & Greed 일일 1회
📈 처리 용량: 시간당 최대 150개 뉴스, 일일 2,500개 데이터 처리

🛠️ 핵심 기술 스택

데이터 수집 레이어

🕷️

Scrapy

고성능 비동기 웹 크롤링 프레임워크. 5개 Spider 동시 실행으로 효율적 데이터 수집.

Python 3.8+
🍜

BeautifulSoup

HTML/XML 파싱 라이브러리. 복잡한 웹페이지 구조에서 정확한 데이터 추출.

bs4 4.11+
⏰

Schedule

Python 작업 스케줄링. Cron 없이 간편한 주기적 크롤링 실행.

schedule 1.1+

빅데이터 처리 레이어

🐘

Hadoop HDFS

분산 파일 시스템. 대용량 원본 데이터를 3중 복제하여 안정적으로 저장.

Hadoop 3.3+
🗄️

MariaDB

오픈소스 RDBMS. 정제된 데이터를 구조화하여 빠른 쿼리 제공.

MariaDB 10.6+

Pandas

데이터 분석 라이브러리. DataFrame으로 효율적인 데이터 변환 및 집계.

Scikit-learn

sklearn 1.3+

🆚 단일 시스템 vs 2-Tier 비교

항목 단일 시스템 2-Tier (CoinTicker)
처리 속도 느림 (순차 처리) 빠름 (분산 병렬 처리)
전력 소비 높음 (PC 24/7 가동) 낮음 (라즈베리파이 20W)
확장성 제한적 무한 확장 가능 (노드 추가)
장애 대응 단일 장애점 분산 구조로 안정성 ↑
비용 높음 (고성능 PC 필요) 낮음 (라즈베리파이 $140)
학습 가치 제한적 높음 (분산 처리 실전 경험)
💡 핵심 인사이트: 2-Tier 아키텍처는 단순히 비용을 절감하는 것이 아니라, 실전 빅데이터 프로젝트에서 요구되는 분산 처리, 고가용성, 확장성 설계 원칙을 모두 담고 있습니다. 이는 취업 포트폴리오로서 매우 높은 가치를 지닙니다.
